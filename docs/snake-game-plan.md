# 🐍 贪吃蛇游戏 — 完整规划书

> 目标：构建一个功能完整、体验流畅、视觉精美的贪吃蛇游戏。
> 技术栈：纯前端 HTML + CSS + JavaScript (Canvas)，零依赖，单文件即可运行。

---

| 功能 | 优先级 | 描述 |
|------|--------|------|
| ✅ 核心玩法 | P0 | 键盘控制蛇移动，吃食物增长，撞墙/撞己游戏结束 |
| ✅ 计分系统 | P0 | 实时显示当前分数、最高分（localStorage 持久化） |

## 二、系统架构

```
┌─────────────────────────────────────────┐
│               HTML 结构                   │
│  ┌───────────────────────────────────┐  │
│  │  标题 / 计分板 / Canvas / 控制栏   │  │
│  └───────────────────────────────────┘  │
├─────────────────────────────────────────┤
│              CSS 样式层                  │
│  深色主题 / 霓虹风格 / 响应式布局        │
├─────────────────────────────────────────┤
│           JavaScript 逻辑层              │
│  ┌──────────┐ ┌──────────┐ ┌────────┐  │
│  │ 游戏状态  │ │ 渲染引擎  │ │ 音效系统│  │
│  │ 管理(SM)  │ │ (Canvas) │ │ (Audio)│  │
│  ├──────────┤ ├──────────┤ ├────────┤  │
│  │ 碰撞检测  │ │ 食物生成  │ │ 输入处理│  │
│  │ +障碍物   │ │ +毒食物  │ │ 键盘+触│  │
│  └──────────┘ └──────────┘ └────────┘  │
└─────────────────────────────────────────┘
```

### 核心模块划分

#### 1. 状态管理模块 (`Game State`)
- 蛇的数据结构：坐标数组 `[{x,y}, ...]`
- 方向：`direction` / `nextDirection`（防止 180° 掉头）
- 食物坐标 + 是否为毒食物标记
- 障碍物数组（超难模式专属）
- 分数、游戏状态（进行中/暂停/结束/胜利）
- 难度参数、速度等级

#### 2. 渲染引擎 (`Renderer`)
- 基于 Canvas 2D
- 网格背景绘制
- 蛇头（发光 + 眼睛）、蛇身（渐变色）
- 食物（正常红球 / 毒食物紫骷髅）
- 障碍物（深灰方块，超难模式）
- 结束/胜利/暂停遮罩层
- `roundRect` polyfill（兼容旧浏览器）

#### 3. 游戏循环 (`Game Loop`)
- `setInterval` 驱动（比 `requestAnimationFrame` 更适合固定 tick 游戏）
- 每次 tick: `update()` → `draw()`
- 速度动态调整：停止旧 interval，启动新 interval

#### 4. 输入处理 (`Input Handler`)
- 键盘：方向键 + WASD
- 触摸：滑动方向识别
- 防 180° 掉头逻辑

#### 5. 音效系统 (`Audio System`)
- Web Audio API（懒初始化）
- 合成器音效：吃食物、吃毒食物、加速、游戏结束、胜利
- 无需外部音频文件

#### 6. 碰撞检测 (`Collision Detection`)
- 墙壁碰撞（普通模式 vs 穿墙模式）
- 自身碰撞
- 障碍物碰撞（超难模式）
- 食物拾取检测（含毒食物检测）

#### 7. 食物生成 (`Food Spawner`)
- 随机坐标，避开蛇身 + 障碍物
- 超难模式：20% 概率生成毒食物
- 若格子几乎占满 → 触发胜利

## 三、数据流

```
用户输入 (键盘/触摸)
       │
       ▼
  handleKey() ──→ 更新 nextDirection
       │
       ▼
  update() (每 tick)
       │
       ├── 1. 应用方向 direction = nextDirection
       ├── 2. 计算新蛇头坐标
       ├── 3. 穿墙模式取模环绕
       ├── 4. 碰撞检测（障碍物/墙壁/自身）
       │      ├── 撞障碍/撞墙(非穿墙) → gameOver
       │      └── 撞己 → gameOver
       ├── 5. 检测是否吃到食物
       │      ├── 毒食物 → gameOver (超难模式)
       │      ├── 正常食物 → 蛇头插入，不删除尾，分数+1
       │      ├── 生成新食物，播放音效
       │      └── 检查速度升级
       └── 6. 调用 draw()
       │
       ▼
  draw() ──→ Canvas 渲染
```

## 四、难度参数详解

| 难度 | baseInterval | speedEvery | speedInc | minInterval | 特殊机制 |
|------|:-----------:|:----------:|:--------:|:-----------:|----------|
| 🟢 简单 | 650ms | 5个 | 30ms | 250ms | 无 |
| 🟡 普通 | 500ms | 5个 | 25ms | 180ms | 无 |
| 🔴 困难 | 350ms | 4个 | 20ms | 100ms | 无 |
| 💀 **超难** | **200ms** | **3个** | **15ms** | **60ms** | **障碍物 + 毒食物** |

### 超难模式专属机制

#### A. 随机障碍物 🧱
- 开局时在地图上生成 **8 个固定障碍物**
- 障碍物不覆盖蛇的初始位置
- 障碍物不覆盖食物
- 蛇撞到障碍物 → 立即 game over
- 障碍物在穿墙模式下仍然致命

#### B. 毒食物 ☠️
- **20% 概率** 生成毒食物（紫色骷髅标记）
- 吃到毒食物 → 立即 game over
- 音效：低沉警示音
- 毒食物不会生成在障碍物上

#### C. 极速体验 ⚡
- 初始速度 200ms（人的反应极限附近）
- 每吃 3 个食物加速 15ms
- 极限速度 60ms（≈ 16.7 步/秒）

## 五、难点与解决方案

| 难点 | 解决方案 |
|------|----------|
| **蛇不能 180° 掉头** | 存储 `direction` 和 `nextDirection`，`update()` 时才应用；校验新旧方向不相反 |
| **穿墙模式** | 碰撞墙壁时取模：`(x + GRID_SIZE) % GRID_SIZE` |
| **蛇身渐变色** | 根据段索引 `idx / snake.length` 计算 RGB 渐变 |
| **食物不生成在蛇身上** | 随机试错，最多 1000 次；若全满则胜利 |
| **障碍物生成避开蛇和食物** | 试错法，将障碍物添加到"禁区"集合中 |
| **毒食物视觉区分** | 紫色径向渐变 + ☠️ 文字标记 |
| **移动端触摸** | 记录 `touchstart` 坐标，`touchend` 时计算滑动方向（忽略 <20px 的误触） |
| **音效兼容** | Web Audio API 需要用户交互后才能激活；懒初始化，try-catch 静默降级 |
| **响应式布局** | 小屏设备容器 padding 自适应，Canvas 保持 400x400 固定分辨率 |
