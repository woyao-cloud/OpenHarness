   六、实施步骤（建议顺序）
     1. 基础设施搭建 — 项目骨架、配置、数据库连接
     2. ORM模型 — SQLAlchemy模型（对应3张表）
     3. CRUD层 — 数据库查询操作
     4. Pydantic Schemas — 请求/响应模型
     5. 通用组件 — Result响应、异常处理、分页
     6. 安全模块 — JWT生成验证、密码哈希、权限依赖
     7. Auth路由+服务 — 登录/注册/刷新/登出
     8. User路由+服务 — 用户CRUD + profile
     9. Role路由+服务 — 角色CRUD
     10. Docker化 — Dockerfile + docker-compose.yml
     11. 数据库初始化脚本 — 复用现有PostgreSQL schema

   七、关键实现要点
     • 统一响应格式: 保留 {code, message, data, timestamp} 格式
     • JWT双Token机制: access_token + refresh_token，与Java版完全对等
     • 权限控制: 通过 Depends(get_current_user) 和 Depends(require_admin) 实现
     • 逻辑删除: SQLAlchemy事件或全局查询过滤器处理 deleted=0
     • 密码: 使用bcrypt，与Java的BCryptPasswordEncoder兼容
     • 分页: 统一分页参数 current / size，返回 {total, pages, current, size, records}
